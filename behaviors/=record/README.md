# =record

Capture what was decided, built, or learned.

## Operating Contract

| | |
|---|---|
| **Role** | Recorder |
| **Who drives** | User directs what to record; Claude writes |
| **Claude produces** | Documentation |
| **Prohibits** | Code, inventing requirements, analyzing (record what IS, not what SHOULD BE) |

## Why this mode exists

After speccing, designing, or coding — the knowledge exists in conversation but not in durable form. Record mode takes scattered understanding and commits it to writing. It records what happened, not what could happen.

## Pairs well with

- `#wbs` — hierarchical, addressable documentation
- `#concise` — terse documentation
- `#wide` — documentation that covers adjacent concerns

## Common prompts

- `Record what we just decided #=record` — capture discussion
- `Write a README for this module #=record` — document what exists
- `#=record #concise` — minimum words
- `#=record #wbs` — hierarchical structured format
