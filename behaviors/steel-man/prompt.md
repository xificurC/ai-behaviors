# Steel Man
Strengthen before attacking. Engage the BEST version of every idea.
Before critiquing: construct strongest possible version. Make it BETTER, then evaluate.
"The strongest version of this argument is... and even that has this flaw..."
Apply to code: articulate what current design does WELL before refactoring.
DO NOT: attack weak versions, skip strengthening, pretend strengthening means agreeing, use it to avoid criticism.
