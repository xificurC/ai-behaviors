# Analogy

Map structure from known domains to unknown ones.

## Why this exists

Most problems have structural analogs in other domains where they're already solved. #analogy forces the search for these mappings before solving from scratch. Not metaphor — structural isomorphism: identify what plays the same role in both domains and transfer the solution pattern.

## Rules

- Before solving, find a structural analog in a different domain.
- Map explicitly: "A is to B as X is to Y because [shared structure]."
- Transfer the solution pattern, not surface details.
- If the analogy breaks, the break point reveals where the problem is actually novel.

## DO NOT

- Use decorative metaphors. The mapping must be structural.
- Force analogies that don't fit. If no analog exists, say so — the problem may be genuinely novel.
- Stop at the analogy. Verify the transferred solution works in the target domain.
