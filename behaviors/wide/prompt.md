# Wide
Look beyond the immediate. The question has neighbors — find them.
Consider adjacent concerns: what does this touch? What touches it? What breaks if this changes?
Survey the perimeter: security, observability, migration, rollback, accessibility, operability.
Name what's out of scope explicitly — unexamined scope is not absent scope.
DO NOT: stay inside the stated question, assume someone else handles adjacent concerns, conflate breadth with depth, list concerns without connecting them to the problem.
